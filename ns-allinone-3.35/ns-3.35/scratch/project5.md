# Project 05: Generate PCAP Files and Analyze Packets in Wireshark

This project demonstrates how to generate **PCAP files** in ns-3 and analyze packet-level network behavior using **Wireshark**.

A PCAP file is a packet capture file. It records the packets that pass through a network device during the simulation.

---

## Network Diagram

```text
Node 0                                      Node 1
UDP Echo Client  ----------------------->  UDP Echo Server
10.1.1.1                                    10.1.1.2

              Point-to-point link
              DataRate = 5Mbps
              Delay    = 2ms
```

---

## Purpose of This Project

The purpose of this project is to understand how to capture and inspect packets generated by an ns-3 simulation.

Using PCAP files, we can verify:

* Whether packets are sent
* Whether packets are received
* Source and destination IP addresses
* UDP port numbers
* Packet size
* Packet timestamps
* Packet direction
* Packet flow between client and server


---

## Important Code Explanation

### 1. Create Two Nodes

```cpp
NodeContainer nodes;
nodes.Create(2);
```

This creates two ns-3 nodes.

```text
Node 0 = UDP Echo Client
Node 1 = UDP Echo Server
```

---

### 2. Configure Point-to-Point Link

```cpp
PointToPointHelper pointToPoint;
pointToPoint.SetDeviceAttribute("DataRate", StringValue(dataRate));
pointToPoint.SetChannelAttribute("Delay", StringValue(delay));
```

This creates a direct link between the two nodes.

```text
DataRate = 5Mbps
Delay    = 2ms
```

The values can be changed from the terminal.

---

### 3. Install Network Devices

```cpp
NetDeviceContainer devices;
devices = pointToPoint.Install(nodes);
```

This installs network devices on both nodes.

A network device is like a network interface card in a real computer.

---

### 4. Install Internet Stack

```cpp
InternetStackHelper stack;
stack.Install(nodes);
```

This installs IP, TCP, UDP, and routing support on the nodes.

Without this, the nodes cannot communicate using IP addresses.

---

### 5. Assign IP Addresses

```cpp
Ipv4AddressHelper address;
address.SetBase("10.1.1.0", "255.255.255.0");

Ipv4InterfaceContainer interfaces;
interfaces = address.Assign(devices);
```

This assigns IPv4 addresses to the nodes.

Usually:

```text
Node 0 = 10.1.1.1
Node 1 = 10.1.1.2
```

---

### 6. Create UDP Echo Server

```cpp
UdpEchoServerHelper echoServer(9);

ApplicationContainer serverApps = echoServer.Install(nodes.Get(1));
serverApps.Start(Seconds(1.0));
serverApps.Stop(Seconds(10.0));
```

This creates a UDP Echo Server on Node 1.

The server listens on UDP port `9`.

---

### 7. Create UDP Echo Client

```cpp
UdpEchoClientHelper echoClient(interfaces.GetAddress(1), 9);
echoClient.SetAttribute("MaxPackets", UintegerValue(maxPackets));
echoClient.SetAttribute("Interval", TimeValue(Seconds(interval)));
echoClient.SetAttribute("PacketSize", UintegerValue(packetSize));
```

This creates a UDP Echo Client on Node 0.

The client sends packets to Node 1.

---

### 8. Enable PCAP File Generation

```cpp
pointToPoint.EnablePcapAll("project5-pcap");
```

This is the most important line in this project.

It tells ns-3 to create PCAP files for packet analysis.

The generated files can be opened in Wireshark.

Example output files:

```text
project5-pcap-0-0.pcap
project5-pcap-1-0.pcap
```

## Generated PCAP Files

After running the simulation, ns-3 creates PCAP files.

Example:

```text
project5-pcap-0-0.pcap
project5-pcap-1-0.pcap
```

Usually:

```text
project5-pcap-0-0.pcap = packets captured at Node 0 interface
project5-pcap-1-0.pcap = packets captured at Node 1 interface
```

---

## Open PCAP File in Wireshark

Use this command:

```bash
wireshark project5-pcap-0-0.pcap
```

or:

```bash
wireshark project5-pcap-1-0.pcap
```

If Wireshark is not installed, install it using:

```bash
sudo apt update
sudo apt install wireshark
```

---

## Useful Wireshark Filters

Show only UDP packets:

```text
udp
```

Show packets from Node 0:

```text
ip.src == 10.1.1.1
```

Show packets from Node 1:

```text
ip.src == 10.1.1.2
```

Show packets going to Node 1:

```text
ip.dst == 10.1.1.2
```

Show UDP Echo packets on port 9:

```text
udp.port == 9
```

Best filter for this project:

```text
udp && udp.port == 9
```

---

## What to Observe in Wireshark

When you open the PCAP file, check these fields:

```text
Source IP
Destination IP
Protocol
Packet length
Source port
Destination port
Time
```

For UDP Echo, you should see packets like:

```text
10.1.1.1  ->  10.1.1.2  UDP
10.1.1.2  ->  10.1.1.1  UDP
```

This means:

```text
Client sends packet to server
Server sends echo reply to client
```

---

## Analyze Packet Delay

You can compare packet timestamps.

Example:

```text
Client sends packet at 2.000000s
Server receives packet at 2.003638s
```

Approximate delay:

```text
2.003638 - 2.000000 = 0.003638s
```

This delay includes transmission time and propagation delay.

---

## Using tcpdump Instead of Wireshark

You can also read PCAP files from the terminal.

```bash
tcpdump -r project5-pcap-0-0.pcap
```

Show only UDP packets:

```bash
tcpdump -r project5-pcap-0-0.pcap udp
```

Show timestamps and IP addresses clearly:

```bash
tcpdump -nn -tt -r project5-pcap-0-0.pcap
```

The UDP Echo Client sends packets from Node 0 to Node 1.
The UDP Echo Server replies back to Node 0.
